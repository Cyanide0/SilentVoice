آلية عمل النظام How the System Works

استعراض تفصيلي لمراحل النظام بدءاً من التقاط الصورة وصولاً إلى تركيب الصوت، مع شرح كل مرحلة وما تؤدّيه. A detailed walkthrough of the system pipeline, from image capture all the way to speech synthesis, with an explanation of each stage.

خطوات النظامSystem Steps

1
التقاط الفيديو عبر الكاميراVideo Capture via Camera
يلتقط النظام تدفق الفيديو الحيّ من كاميرا المستخدم أو من مقاطع فيديو مسجّلة مسبقاً. تُعالَج كل إطار على حدة بمعدّل لا يقلّ عن ٣٠ إطاراً في الثانية لضمان سلاسة الكشف. The system captures a live video stream from the user's camera or from pre-recorded video clips. Each frame is processed individually at a rate of at least 30 frames per second to ensure smooth detection.
Webcam 30fps تدفّق مباشرLive Stream
2
المعالجة الأوّلية بـ OpenCVPre-processing with OpenCV
تُعالَج الإطارات باستخدام OpenCV لتحسين جودة الصورة؛ يشمل ذلك تغيير الحجم، التطبيع، إزالة الخلفية، وتعديل الإضاءة. هذه الخطوة تُهيّئ الصورة لمرحلة الكشف الدقيق عن اليدين. Frames are processed using OpenCV to improve image quality, including resizing, normalization, background removal, and lighting adjustment. This step prepares the image for the precise hand detection stage.
Resize Normalize Augmentation إزالة الخلفيةBackground Removal
3
تتبّع نقاط اليد بـ MediaPipeHand Landmark Tracking with MediaPipe
يستخدم النظام MediaPipe Hands لرصد وتتبّع ٢١ نقطة معلَم على كل يد. تُمثّل هذه النقاط مفاصل وأطراف الأصابع وتُوفّر بيانات إحداثيات ثلاثية الأبعاد تُشكّل مدخل النموذج الأساسي. The system uses MediaPipe Hands to detect and track 21 landmark points on each hand. These points represent finger joints and tips, providing 3D coordinate data that forms the model's primary input.
21 Landmark ثلاثي الأبعاد3D Coordinates MediaPipe Hands
4
التعرّف بنماذج LSTMLSTM Models
تُحلّل LSTM التسلسل الزمني للإشارة عبر الإطارات المتتالية. يُستخدم التعليم المسبق والضبط الدقيق لتحسين الدقة مع البيانات المحدودة. While LSTM analyzes the temporal sequence of the sign across consecutive frames. Pre-training and fine-tuning are used to improve accuracy with limited data.
LSTM Transfer Learning Fine-tuning PyTorch
5
تحويل الإشارة إلى نصّConverting Sign to Text
يُرسَل ناتج النموذج إلى طبقة التحويل التي تُقابل الإشارة المُكتشَفة مع قاموس الكلمات المناظرة. تُبنى الجمل باستخدام نموذج لغوي بسيط يجمع الكلمات في تسلسل منطقي. The model's output is passed to a conversion layer that maps the detected sign to its corresponding word dictionary entry. Sentences are constructed using a simple language model that arranges words in a logical sequence.
Word Mapping بناء الجملةSentence Building Label Encoding
6
تركيب الصوت والمخرجات النهائيةSpeech Synthesis & Final Output
تُحوَّل الجملة النصية إلى صوت باستخدام مكتبة gTTS أو Azure Speech API، مما يُتيح التفاعل الشفهي الكامل. يُعرَض النص والصوت معاً في واجهة المستخدم لضمان إمكانية الوصول الشامل. The text sentence is converted to speech using the gTTS library or Azure Speech API, enabling full verbal interaction. Both text and audio are displayed together in the user interface to ensure universal accessibility.
gTTS Azure Speech API TTS مخرجات مزدوجةDual Output
منهجية التطوير — نموذج الشلّال Development Methodology — Waterfall Model
يعتمد المشروع على منهجية الشلّال (Waterfall) لكونها تناسب المشاريع ذات المتطلبات المحدّدة مسبقاً، إذ تكتمل كل مرحلة توثيقاً وتحقيقاً قبل الانتقال إلى التالية. The project adopts the Waterfall methodology as it suits projects with pre-defined requirements, where each phase is fully documented and verified before moving to the next.
